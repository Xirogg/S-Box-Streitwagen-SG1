# Tool-Stack & Zuständigkeiten

Leitprinzip: **„eine Aufgabe – ein Tool – eine Zuständigkeit"** (Research Bible Producing, Würfl, 2026).

| Kategorie | Tool | Zweck | Verantwortung |
|---|---|---|---|
| Projektmanagement | **Codecks** | Cards/Decks, Milestones, Runs, Hero Cards | Producing |
| Boards & Brainstorming | **Miro** | Vision Board, Workshops, Whiteboarding | Producing + alle |
| Kommunikation | **Discord** | Daily Sync, Voice, Channels (#art, #code, #design) | Producing (Moderation) |
| File-Sharing & Repo | **GitHub** | Code-Repo, Asset-Distribution, Issues | Programming |
| Engine | **s&box (Source 2)** | Spielbau, Scenes, Hammer, Hot-Reload | Programming |
| GDD | **Codecks (GDD-Space)** | Asset-Decks + lebende Dokumentation | Game Design |
| Bug-Tracking | **Codecks (Bug-Deck)** | Triage, Reproduktion, Status | Producing + Reporter |
| Stakeholder-Pitches | **PowerPoint / Keynote** | Pitch-Decks, Status-Updates | Producing |
| **Wissensmanagement** | **Obsidian** *(neu)* | Vernetzte Doku, MOCs, Templates, Git-Sync | alle |

Tool-Quellen: Codecks (o. J.); Carter (2024); Facepunch Studios (o. J.); SubZero Studios (2025).

## Obsidian-Einordnung *(Ergänzung)*

Obsidian wurde im bisherigen Tool-Stack der Research Bibles **nicht** geführt. Es ergänzt den Stack als lokal-first, Markdown-basiertes Wissensmanagement und nutzt dieselbe GitHub-Versionierung wie das Programming-Department (Obsidian, o. J.; Vinzent03, o. J.).

- Vorteil ggü. reiner Codecks-Doku: vernetzte Notizen (`[[Wikilinks]]`), Graph-Ansicht, plain-text in Git lesbar.
